# 🐾 업캐스팅 / 다운캐스팅 과제 (Animal)

---

## 🔹 과제 목적
- 업캐스팅(Upcasting)과 다운캐스팅(Downcasting)의 개념을 이해한다.
- 상속 관계에서의 다형성과 오버라이딩 동작을 설명할 수 있다.
- `instanceof` 연산자의 역할을 이해한다.

---

## 📌 과제 설명

본 과제는 `Animal` 클래스를 부모 클래스로 하고,  
`Dog` 클래스를 자식 클래스로 하는 상속 구조를 기반으로  
업캐스팅과 다운캐스팅의 개념을 **코드와 글로 설명**하는 과제이다.

---

## 📌 클래스 구성 조건

### 1️⃣ 부모 클래스 : Animal

- 필드
    - name
    - age
- 모든 필드는 `private`으로 선언할 것
- 생성자는 모든 필드를 초기화할 수 있도록 정의할 것
- Getter / Setter를 구현할 것
- `showInfo()` 메서드를 call1() 타입으로 정의할 것  
  (이름과 나이를 출력하도록 구현)

---

### 2️⃣ 자식 클래스 : Dog

- `Animal` 클래스를 상속받을 것
- 자식 고유의 필드를 최소 2개 이상 정의할 것
    - 예: 품종, 예방접종 여부 등
- 생성자에서 부모 클래스의 생성자를 호출할 것
- `showInfo()` 메서드를 오버라이딩하여  
  부모 정보 + 자식 정보를 모두 출력하도록 구현할 것

---

## 📌 Main 클래스 요구 사항

### 1️⃣ 업캐스팅

- `Dog` 객체를 생성하여 `Animal` 타입 변수에 저장할 것
- 업캐스팅된 객체를 통해 `showInfo()`를 호출할 것
- **왜 자식 클래스의 메서드가 실행되는지 주석으로 설명할 것**

---

### 2️⃣ 다운캐스팅

- 업캐스팅된 객체를 다시 `Dog` 타입으로 변환할 것
- 다운캐스팅 전에 반드시 객체 타입을 확인할 것
- 다운캐스팅 후 자식 클래스의 고유 메서드 또는 필드를 사용할 것
- **객체 타입 확인을 하지 않을 경우 발생할 수 있는 문제를 주석으로 설명할 것**

---

### 3️⃣ 다운캐스팅이 불가능한 경우

- `Animal` 타입의 객체를 하나 더 생성할 것
- 해당 객체를 `Dog` 타입으로 다운캐스팅하지 말 것
- **왜 해당 객체는 다운캐스팅이 불가능한지 주석으로 설명할 것**

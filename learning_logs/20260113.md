# 제네릭 와일드 카드
## 제너릭
- 클래스, 인터페이스 함수에 사용되는 자료형을 매개변수화 할 수 있는 문법이고 다양한 타입에서 재사용이 가능
```java
public class Box<T>{
    // 필드 변수 선언
    private T value;

    //getter setter
    public T getValue(){
        return value;
    }
    public void setValue(T value){
        this.value = value;
    }
}
```
이상의 방식으로 작성했을 때 value의 자료형을 고정시키지 않고 사용 가능함.
자바 자체의 자료형(class) 뿐만 아니라 개발자가 정의한 것도 집어 넣을 수 있음
백엔드 설계에서 매우 중요함

## 와일드카드
- 이상의 제네릭 타입을 사용할 때 구체적인 타입을 명시하지 않고 범용적으로 처리하고 싶을 때 사용함. '?' 사용

```java
public static void main(String[] args) {
    List<?> examplelist = new ArrayList<String>();
}
```

### 종류
1. 모든 타입 허용
   - 읽기 전용임 어떤 타입이든지 일단 받아놓고 분류해야 할 때 
2. T 또는 T의 서브 타입(자식 클래스)
   - 읽기 전용임 제한된 타입에서 읽을 때
3. T 또는 T의 슈퍼 타입(부모 클래스)
   - 쓰기 전용임 제한된 타입으로 넣을 때
